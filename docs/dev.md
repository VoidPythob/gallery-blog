# 画廊博客开发文档

## 1. 项目简介

本项目是一个静态博客站点，目标是同时支持以下内容展示：

- 画廊作品展示（绘画内容）
- 博文发布与阅读
- 基于标签的内容过滤
- 博主信息与友情链接展示
- 主题系统（亮色/暗色、主题色、滤镜、阴影、圆角）
- 每日一言与公告
- 时间轴视图

## 2. 技术选型

- 包管理：`pnpm`
- 前端框架：`Vue 3 + TypeScript`
- 样式方案：`Tailwind CSS`
- 动画库：`Animate.css`
- 路由管理：`vue-router`
- 状态管理：`Pinia`
- Markdown 渲染：`markdown-it`
- 组件库：`TDesign`
- 资源存储：`OSS`

### 2.1 补充约束

- 组件库按需引入，避免全量打包
- `OSS base_url` 必须通过外部配置注入，不得写入仓库

## 3. 功能说明

### 3.1 博文系统

#### 3.1.1 数据结构

```ts
type Article = {
  title: string;
  cover_url: string;
  tags: string[];
  isToped: boolean;
  readTime: number; // 单位：秒
  introduction: string;
  markdown: string;
};
```

#### 3.1.2 存储规则

- 路径：`public/posts/articles`
- 文件名：使用 `uuid`
- 格式：`json`

#### 3.1.3 搜索

- 按标题做模糊搜索
- 需要建立索引优化搜索性能

#### 3.1.4 标签

- 一篇博文可绑定多个标签

#### 3.1.5 置顶

- `isToped = true` 的博文归入置顶集合
- 置顶集合按时间顺序排序（建议明确为倒序：最新优先）

#### 3.1.6 博文卡片视图规范

- 卡片使用圆角边框
- 背景色、字体色随主题变化
- 悬停时有轻微 3D 抬升效果，并改变背景色
- 内容区使用垂直 `flex` 布局
- 标题位于最上方
- 元数据区域使用 `icon + 内容`，并用 `|` 分隔
- 有封面时，标题和元数据覆盖在封面区域上
- 简介超出显示省略号
- 时间使用 24 小时制（`HH:mm`）
- 阅读时长自动选择单位（秒/分钟）
- 标签使用独立组件，并随主题色调整背景
- 标签区布局：左侧 icon，右侧标签列表
- 置顶标签样式特殊，且始终展示在标签首位

#### 3.1.7 博文列表布局

- 使用瀑布流布局
- 布局尽量紧凑
- 每页固定 10 条数据

#### 3.1.8 响应式

- 屏幕宽度 `< 650px` 时，博文卡片切换为垂直 `flex` 布局

### 3.2 标签视图

#### 3.2.1 功能目标

- 提供按标签聚合的内容视图
- 在同一页面中展示该标签下的文章或图片（画廊项）
- 支持手动切换展示模式

#### 3.2.2 展示与切换

- 页面顶部显示当前标签名称与该标签的内容总数
- 提供模式切换按钮组：
- `文章模式`：仅显示该标签下的博文
- `图片模式`：仅显示该标签下的画廊图片
- 默认模式可配置（建议默认 `文章模式`）
- 切换模式时仅切换数据视图，不跳转页面

#### 3.2.3 数据与筛选规则

- 标签筛选为精确匹配（tag 完全一致）
- 同时支持来自博文与画廊的标签数据
- 当某模式下无数据时，展示空状态文案

#### 3.2.4 视图复用

- `文章模式` 复用博文卡片样式与分页规则
- `图片模式` 复用画廊卡片样式与分页规则
- 保持统一主题能力（亮暗色、主题色、圆角、阴影）

#### 3.2.5 分页与响应式

- 各模式独立分页（互不影响当前页码）
- 响应式断点与主站一致：`< 650px` 时切换为移动端紧凑布局

### 3.3 通用分页组件

- 使用组件库现成分页组件
- 支持页码跳转输入
- 分页按钮样式为全圆角
- 包含左右切换按钮
- 分页组件位于数据视图下方，与内容垂直排列

### 3.4 导航系统

#### 3.3.1 路由项

- 首页
- 博文
- 画廊
- 友情链接

#### 3.3.2 布局与行为

- 导航分为侧边栏与顶部导航栏
- 屏幕宽度 `< 650px` 时隐藏侧边栏，启用顶部导航
- 顶部导航布局：左侧站点名，中间路由，右侧搜索
- 顶部导航支持折叠菜单
- 主内容区需有内边距（padding）
- 导航与内容区之间需有合理间距（gap）
- 每日一言面板与侧边导航采用上下布局

### 3.5 每日一言

#### 3.4.1 存储

- 路径：`public/posts/everyday-words.json`
- 结构：字符串列表

#### 3.4.2 选择逻辑

- 每天随机选择一条
- 使用“日期”作为随机因子，保证同一天结果固定

### 3.6 公告系统

#### 3.5.1 存储

- 路径：`public/posts/announcement.json`
- 结构：公告列表
- 当前阶段可先按单条公告渲染

### 3.7 时间轴

- 提供画廊与博文的时间轴展示

```ts
type TimelineItem = {
  date: string; // 日期
  targetLink: string; // 跳转链接
};
```

### 3.8 主题系统

- 支持亮色/暗色切换
- 支持主题色配置
- 支持滤镜配置
- 支持圆角配置
- 支持阴影配置

## 4. 开发建议（可选）

- 统一类型定义到 `src/types`，避免重复声明
- 搜索索引可在构建阶段生成，减少运行时开销
- 主题配置建议集中管理为全局 `token`（颜色、圆角、阴影、滤镜）
- 数据文件增加基础校验，避免运行时字段缺失
